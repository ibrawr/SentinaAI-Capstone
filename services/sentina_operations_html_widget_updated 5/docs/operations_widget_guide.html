Senti role-based widget

This widget supports operations, sustainability, and exhibitor flows inside the same shell. The launcher label, assistant title, actions, and analytics flow all change by role.

Roles

  • Senti Operations: venue operations monitoring and drill-downs
  • Senti Sustainability: energy, comfort, efficiency, and by-event analysis
  • Senti Exhibitor: assignment-based booth analytics and exhibitor comparisons

Exhibitor rules

  • scope is assignment-based
  • date range is locked to the event window
  • comparison is intentional and event-aware
  • comparison is not repeated as a follow-up on every result card

Sustainability by event overview

By Event Overview computes event-level metrics from the dataset and event mapping file. It can show energy, carbon, occupancy, flow, queue share, overcrowded share, and top-event rankings for the selected range.

Saved views

Saved views preserve the request filters and the rendered result snapshot used in the prototype.

Prototype notes

This package uses local CSV data and a FastAPI backend for the prototype. It is structured so the frontend team can later replace the CSV layer with the real backend API and database architecture.
